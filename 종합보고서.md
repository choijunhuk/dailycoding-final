본 프로젝트는 AI를 활용한 난이도별 문제 생성 기능과 코드 리뷰 기능을 중심으로 코딩 학습 플랫폼의 핵심 구조를 구현하는 것을 목표로 진행되었다. 현재
  난이도별 문제 생성 로직과 코드 리뷰 로직은 일정 수준까지 구현이 완료되어, 플랫폼의 주요 기능으로 동작할 수 있는 기반이 마련된 상태이다. 이를 통해 사
  용자는 자신의 수준에 맞는 문제를 추천받고, 제출한 코드에 대한 피드백을 확인할 수 있도록 설계되었다.
  이와 함께 플랫폼의 차별화 요소로서 1대1 코딩테스트 대결 시스템 도입이 추진되었다. 해당 기능은 두 사용자가 동일한 문제를 제한 시간 내에 해결하고, 제
  출 결과를 비교하여 승패를 가리는 방식으로 구성된다. 프론트엔드에서는 대결방 생성 및 입장, 문제 확인, 코드 제출, 결과 확인 기능을 구현하였고, 백엔드
  에서는 대결방 정보 관리, 문제 할당, 제출 결과 저장, 채점 결과 반환 등의 로직을 설계하였다. 사용자가 코드를 제출하면 서버가 채점 API와 연동하여 실행
  결과를 받아오고, 이를 바탕으로 정답 여부와 대결 진행 상황이 반영되도록 구현하였다.
  서비스를 실제 환경에서 운영하기 위해 초기에는 ngrok을 활용한 외부 테스트를 진행하였으며, 이후 프론트엔드는 Netlify, 백엔드는 Railway에 배포하는 방식
  으로 전환하였다. 다만 Railway의 무료 크레딧 한계로 인해 일정 기간이 지나면 서버가 자동으로 중지되는 문제가 발생하였고, Railway 환경에서는 Docker in
  Docker를 지원하지 않아 도커 기반 채점 샌드박스 운영에도 제약이 있었다. 이에 따라 도메인 구매 후 Hetzner VPS를 활용하여 직접 서버를 운영하는 방안을
  검토하고 있으며, 보다 안정적인 서비스 운영 환경으로의 전환 가능성을 모색하고 있다.
  현재 채점 시스템은 Piston API를 활용하여 운영되고 있다. 본래 목표는 Docker를 이용한 독립적인 샌드박스 환경을 구축하는 것이었으나, 현 단계에서는
  Piston API가 제공하는 실행 환경을 활용하여 채점 기능을 대체하고 있다. 향후 VPS 기반 환경으로 이전할 경우, 채점 시스템 역시 보다 안정적이고 독립적인
  구조로 확장하는 방안이 검토 대상이다.
  부가 기능으로는 Gmail SMTP를 활용한 이메일 인증 시스템을 도입하여 사용자 계정의 신뢰성과 보안성을 강화하였다. 또한 Stripe를 기반으로 구독 모델 구현
  도 진행하였다. 다만 구독 기능은 결제 연동 자체는 구현되었으나, 구체적인 가격 정책과 제공 기능 구성이 아직 확정되지 않아 실제 서비스 적용 단계에서는
  미완성에 가까운 상태이다. 따라서 향후에는 구독 등급별 제공 기능, 가격 체계, 사용자 유인 요소를 구체화하는 작업이 추가적으로 필요하다.
  종합적으로 본 프로젝트는 AI 기반 문제 생성 및 코드 리뷰 기능, 1대1 코딩테스트 대결 기능, 이메일 인증 및 구독 모델 등 플랫폼의 핵심 요소를 단계적으로
  구축해 왔으며, 현재는 배포 안정성과 채점 인프라 고도화를 포함한 운영 환경 개선이 주요 과제로 남아 있다.
